# 要件定義書

# ProjectFlow Pro

## 1. プロジェクト概要

### 背景

AI開発や受託開発では、
クライアントの要望整理・ヒアリング・要件定義・提案・進捗管理・納品確認など、
実装前後に整理すべき情報が多い。

しかし実際には、

- 要件が曖昧なまま実装を始めてしまう
- 機能を増やしすぎて方向性がぶれる
- 「やること」「やらないこと」が整理されていない
- 提案内容やPhase2案が記録に残らない
- 納品後に追加要望が来ても、前回内容を振り返りづらい

という課題がある。

そこで、
案件の進行をテンプレート形式で整理し、
ヒアリングから納品・振り返りまで一連で管理できるツールを作成する。

---

## 2. 目的

受託・AI開発案件において、

- ヒアリング漏れ
- 要件のブレ
- MVP不明確化
- 実装途中の方向転換

を防ぎ、
案件進行を整理・記録・振り返りしやすくする。

---

## 3. ターゲットユーザー

- AI開発初心者
- Claude Code利用者
- 個人開発者
- 副業エンジニア
- 受託開発初心者
- 要件整理に不安がある人

---

## 4. MVPスコープ

### 含むもの

- クライアントブリーフ入力
- ヒアリング質問テンプレート
- Why / Who / What / How 整理
- 要件整理
- 提案書メモ
- WBS作成
- 実装進捗メモ
- 納品チェックリスト
- 振り返りメモ
- メールテンプレコピー
- localStorage保存
- レスポンシブ対応

### 含まないもの

- ログイン機能
- 複数案件管理
- データベース保存
- AI自動生成
- PDF出力
- チーム共有機能

## 7. 機能要件

| ID | 機能名 | 内容 | 優先度 | Phase |
|---|---|---|---|---|
| F-01 | クライアントブリーフ入力 | 案件名・クライアント名・依頼内容などを入力できる | 高 | MVP |
| F-02 | ヒアリング質問管理 | Why / Who / What / How の質問を入力・編集できる | 高 | MVP |
| F-03 | 要件整理 | やること / やらないこと / Phase2 を整理できる | 高 | MVP |
| F-04 | 提案書メモ | 課題・解決策・技術構成・費用などを整理できる | 高 | MVP |
| F-05 | WBS作成 | 作業内容・日数・進捗を管理できる | 高 | MVP |
| F-06 | 実装進捗メモ | やったこと・次やること・困りごとを記録できる | 中 | MVP |
| F-07 | 納品チェックリスト | 納品前確認項目をチェックできる | 高 | MVP |
| F-08 | 振り返りメモ | 技術面・PM面・改善点を記録できる | 中 | MVP |
| F-09 | メールテンプレ表示 | ヒアリング依頼・提案送付などのテンプレを表示できる | 中 | MVP |
| F-10 | コピー機能 | テンプレ文をコピーできる | 中 | MVP |
| F-11 | localStorage保存 | 入力内容をブラウザに保存する | 高 | MVP |
| F-12 | レスポンシブUI | スマホでも操作できるUI | 高 | MVP |
| F-13 | PDF出力 | 案件カルテをPDF保存できる | 中 | v1.0 |
| F-14 | Markdown出力 | requirements.mdや提案書を出力できる | 低 | v1.0 |
| F-15 | ログイン機能 | ユーザー認証を行う | 低 | v2.0 |
| F-16 | 複数案件管理 | クライアントごとに案件保存できる | 低 | v2.0 |
| F-17 | AI質問生成 | AIでヒアリング質問を生成する | 低 | v2.0 |

## 9. UI方針
PC・スマホ両対応
サイドバーで各ステップを切り替え
入力内容はリアルタイム保存
シンプルで落ち着いたUI
長文入力しやすい設計
非エンジニアでも理解しやすい表現
ボタンサイズはスマホ操作を考慮
進捗状況が分かるUIを入れる
