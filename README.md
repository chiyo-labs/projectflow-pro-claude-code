# ProjectFlow Pro

AI-powered workflow tool for freelance engineers.

受託開発時の「ヒアリング → 要件定義 → MVP設計 → 見積 → 請求」までを、一つの画面で整理・管理できるAI支援ツールです。

Claude Code を活用した実務フローを意識して開発しています。

## Features

### ヒアリング整理
- クライアント情報・依頼概要・予算・納期を整理
- AIによるヒアリング質問生成

### 要件定義
- 「やること / やらないこと / Phase2」を整理
- 実装スコープを明確化

### 提案書メモ
- 課題
- 解決策
- 技術構成
- スケジュール
- 費用感

をまとめて管理

### MVP設計
- 機能を MVP / v1.0 / v2.0 に分類
- Claude Code 用の実装指示を自動生成

### 見積書作成
- AIによる見積項目生成
- 価格モード切り替え
  - 駆け出し
  - 標準
  - 強気
- 消費税対応
- PDF出力

### 請求書作成
- 見積内容を請求書へ反映
- PDF出力
- 振込先情報の表示

### 運用チェック（運用・引き継ぎ）

LINE bot / Vercel / Supabase / LINE Developers などの外部サービスを使う案件向けに、実装だけでなく「運用責任の整理」を行うチェックリスト。

| カテゴリ | 目的 |
|---|---|
| アカウント権限チェック | Vercel / Supabase / LINE Developers / GitHub などの招待・権限確認 |
| 作業同意チェック | 作業範囲の書面合意・追加費用条件・本番環境影響の事前説明 |
| 環境変数・APIキー管理 | `.env` の Git 除外、本番/開発キー分離、管理責任者の確認 |
| 納品後の管理者確認 | 管理者アカウント引き渡し、請求情報のクライアント名義変更 |
| 保守範囲の整理 | 納品後に対応する範囲・しない範囲を自由記述で明記 |
| クライアント引き継ぎ | 操作説明書・APIキー管理方法・今後の費用感の伝達確認 |

各項目には1行メモ欄があり、「誰に確認したか」「確認日」「未対応の理由」などを記録できます。

## How to Use

### 1. クライアントブリーフを入力
案件名、クライアント名、依頼内容、予算、納期を入力します。

### 2. ヒアリング質問を作成
AI生成ボタンを使うと、Why / Who / What / How に分けた質問案を作成できます。

### 3. 要件整理
ヒアリング回答をもとに、
「やること」「やらないこと」「Phase 2」を整理します。

### 4. 提案書メモを作成
課題、解決策、技術構成、スケジュール、費用感をまとめます。

### 5. MVP設計
機能を MVP / v1.0 / v2.0 に分類し、Claude Codeへ貼り付ける実装指示を作成します。

### 6. 見積書を作成
AIで見積項目のたたき台を生成し、金額を調整できます。
作成した見積書はPDF出力できます。

### 7. 請求書を作成
見積書の内容を読み込み、請求書を作成できます。
請求書もPDF出力できます。

### 8. 運用チェックを進める
外部サービスを使う案件では「運用・引き継ぎ」セクションの「運用チェック」を活用します。

| タイミング | 確認内容 |
|---|---|
| 実装前 | アカウント権限チェック・作業同意チェック |
| 実装中 | 環境変数・APIキー管理 |
| 納品前後 | 納品後の管理者確認・保守範囲の整理・クライアント引き継ぎ |

全項目にチェックが入ると、進捗バーが緑になり完了が一目で確認できます。

### 9. 進捗・納品・振り返り
WBS、実装進捗、納品チェックリスト、振り返りを記録できます。

## Purpose

受託開発を進める中で、

- ヒアリング内容が散らばる
- 要件整理が曖昧になる
- 見積作成に悩む
- Claude Code への指示が毎回ブレる

という課題を感じ、
実際の受託フローを整理するために開発しました。

AIを単なる文章生成ではなく、
「案件進行を支援するツール」として活用することを目的としています。

また、LINE bot / Vercel / Supabase / LINE Developers のような外部サービスを使う案件では、実装完了だけでなく「アカウント権限の整理」「作業同意の確認」「納品後の管理責任の明確化」が実務上のトラブル回避に直結します。
「運用チェック」機能はこうした運用面のリスクを、実装フローの中で見落とさないために追加しました。
